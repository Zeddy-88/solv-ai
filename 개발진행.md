## 🔄 Phase 2 — PDF 업로드 연동 및 AI 데이터 확장 (진행 중)

### 2.1 PDF 업로드 모듈

- [ ] **드래그 앤 드롭 업로드 UI**
  - 홈 화면 또는 사이드바 "새로운 재무분석" 버튼 클릭 시 표시
  - 파일 선택 또는 드래그 앤 드롭 (50MB 제한)
  - 업로드 진행 상태 바 (%) 표시
- [ ] **파일 유효성 검사**
  - PDF 타입 검증 (`application/pdf`)
  - 파일 크기 제한 (50MB)
  - 페이지 수 확인 (최대 100페이지 권장)
- [ ] **업로드 피드백 UI**
  - 성공: 분석 시작 로딩 스크린
  - 실패: 에러 메시지 + 재시도 버튼

### 2.2 Gemini 2.5 Flash API 연동

- [ ] **백엔드 API 엔드포인트** (`/api/analyze`)
  - PDF Base64 변환 후 Gemini API 전송
  - 마스터 프롬프트 적용 (4인 페르소나 + 6개 제안 + 스크립트)
  - JSON 응답 파싱 및 유효성 검사
- [ ] **로딩 상태 UI**
  - 분석 중 스켈레톤 카드 표시
  - "AI가 재무제표를 분석하고 있습니다..." 진행 메시지
  - 예상 소요 시간 표시 (약 30초~2분)
- [ ] **에러 처리**
  - API 오류 (500): "분석 서버 오류. 잠시 후 다시 시도해주세요."
  - 파싱 오류: "PDF를 읽을 수 없습니다. 다른 파일을 시도해보세요."
  - 타임아웃: 2분 초과 시 재시도 옵션

### 2.4 동적 데이터 바인딩

- [ ] **정적 Mock 데이터 → AI 생성 데이터로 교체**

## 📋 Phase 3 — PDF 보고서 생성 + 이력 공유 (예정)

### 3.1 대표이사 제출용 PDF 자동 생성

- [ ] **보고서 템플릿 설계**
  - Solv AI 브랜드 헤더 (Klein Blue)
  - 기업 기본 정보 페이지
  - 핵심 지표 + 재무 진단
  - 3개년 추이 차트 (막대 또는 선그래프)
  - 제안 항목 요약 (6개, 영업 전략 멘트 제외)
  - 기대 효과 표
- [ ] **생성 엔진 선택**
  - 옵션 A: jsPDF + html2canvas (클라이언트 사이드)
  - 옵션 B: Puppeteer (서버 사이드, 품질 우수)
- [ ] **PDF 생성 버튼 기능 활성화**
  - 현재 UI만 있는 버튼에 실제 기능 연결
  - 다운로드 진행 상태 표시

### 3.2 분석 이력 관리 시스템

- [ ] **이력 저장 (로컬/서버)**
  - localStorage (단기) / Supabase DB (장기)
  - 이력 항목: 회사명, 분석 일시, 주요 수치
- [ ] **이력 목록 UI 기능화**
  - 사이드바 "최근 분석 이력" 실 데이터 연동
  - 클릭 시 해당 분석 결과 복원
  - 날짜 그룹화 (오늘, 어제, 이번 주)
- [x] **즐겨찾기 기능**
  - [x] 별표 클릭 시 즐겨찾기 등록/해제
  - [x] 즐겨찾기 이력 상단 별도 섹션 상시 표시

### 3.3 분석 이력 공유

- [ ] **공유 링크 생성**
  - "분석 이력 공유" 버튼 클릭 시 고유 링크 생성
  - 링크 복사 기능 (클립보드)
  - 유효 기간 설정 (7일 / 30일 / 영구)
- [ ] **공유된 뷰어**
  - 공유 링크 접속 시 읽기 전용 뷰어
  - 영업 스크립트 숨김 처리 옵션
